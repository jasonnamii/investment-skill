# Investment Skill — 인베스트먼트 스킬

스타트업 투자 전 생애주기 지식 백과 + 진단/의사결정 엔진.
7L(Pre-Seed→Secondary) × 3V(전략·법률·재무) + 5X(지분·거버넌스·글로벌·실패·관계) + VX(밸류에이션).

## §2. 모드 자동 판별

| 입력 유형 | 모드 | 스포크 로딩 | 출력 |
|-----------|------|-----------|------|
| 현재 상황 설명 ("우리 지금 시리즈A 준비 중") | **진단** | 핑퐁 P1→단계판별→해당 L축+관련 X축 | 갭 분석 + 체크리스트 + 액션 |
| 목표·방향 질문 ("밸류에이션 어떻게 올려") | **전략** | 해당 L축 + VX + 교차분석 | 조건부 전략(유효/무효) + 대조군 + 액션 |
| 갈림길 ("팔까 다음 라운드 갈까") | **판정** | 양쪽 L축 + X4 + X5 + 교차분석 | 리스크 매트릭스 + 핵심변수 + 재평가기한 |
| 개념·정보 질문 ("안티딜루션 종류") | **조회** | 해당 스포크 1개만 | 직접 정보 응답 + 관할권 태그 + 관련 대조 패턴 1쌍 |

**조회모드 경량화:** 조회모드('~알려줘')는 핑퐁 스킵 → 해당 스포크 직접 로드. L축×X5축 중복 로딩 방지: L단계 스포크 내 '투자자 관점' 섹션이 X5와 중복 시, L단계 스포크의 해당 섹션만 참조 (X5 별도 로드 금지).

복합 시 진단→전략 파이프라인. 애매하면 사용자에게 모드 확인.

## §3. 핑퐁 (진단·전략·판정 공통)

### Phase 구조

| Phase | 질문 범위 | 문항 수 |
|-------|----------|---------|
| P1 현황 | 업종, 단계, 팀 규모, 매출/트랙션, 현재 캡테이블 | 3~5 |
| P2 목적 | 투자유치 목적, 목표 금액, 희망 밸류, 타임라인 | 2~3 |
| P3 제약 | 기존 투자자 관계, 법적 제약, 지분 구조 이슈 | 1~3 |

### 핑퐁 실행 규칙

1. P1 완료 → 단계 판별 (L1~L7) → 해당 L스포크 + 관련 X축 로드
2. Phase당 1~3문항씩 묶어 질문 (한 번에 전부 쏟지 않음)
3. 답변에 따라 불필요 질문 자동 스킵
4. P1에서 단계 판별 불가 → 추가 질문 1회. 2회차에도 모호 → 가장 유사 단계로 잠정 배정 + 명시
5. 조회 모드는 핑퐁 스킵 → 즉시 해당 스포크 로드 후 응답

## §4. 분석 파이프라인

### 진단 모드

①상황 구조화(업종·단계·규모·이슈·제약) → ②스포크 로드(L축+X축) → ③체크리스트 대조(해당 단계 필수 항목 vs 현재 상태) → ④갭 분석(미충족 항목 식별) → ⑤패턴 대조(`pattern-index.md`에서 해당 단계 성공/실패 쌍 검색 → 갭 항목별 대조 패턴 병치, 절대규칙 #5 적용) → ⑥교차분석 참조(모순·강화·빈자리 중 해당건) → ⑦액션 제시(우선순위별)

### 전략 모드

①목표 확인 → ②`pattern-index.md` 마스터 테이블에서 후보 패턴 스캔 → ③해당 스포크 로드 → ④패턴 라이브러리 섹션에서 정규화 패턴 매칭(성공 패턴 + 대조 실패) → ⑤조건부 전략 도출:

```
전략: [구체적 전략]
유효 조건: [작동하는 상황]
무효 조건: [역효과 나는 상황]
근거 패턴: [성공 사례 핵심]
대조 패턴: [실패 사례 핵심]
```

⑥우선순위(임팩트×실행가능성) → ⑦리스크 체크 → ⑧액션 플랜

### 판정 모드

①갈림길 구조화(A vs B) → ②`pattern-index.md`에서 양쪽 관련 패턴 식별 → ③해당 스포크 로드 → ④양쪽 패턴 라이브러리에서 정규화 패턴 매칭 → ⑤핵심 변수 식별("성패를 가른 변수") → ⑥리스크 매트릭스:

| 선택지 | 최선 | 최악 | 핵심 변수 | 되돌림 가능성 |

⑦판정 보조(데이터 가리키면 방향 제시, 불충분하면 필요 추가정보 명시) → ⑧**재평가 기한 필수**(3/6/12개월 + 트리거 변수)

## §7. 관찰한계 (스킬 내 선언)

| 한계 | 영향 | 대응 |
